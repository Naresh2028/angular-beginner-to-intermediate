# COMPONENTS

## Definition
A Component is the smallest self-contained UI unit in Angular, combining template (view) and logic (behavior) into one structured block.

Short memory version:

- Component = View + Behavior.

## Analogy
A component is like a LEGO block.

- Each block has its own shape.

- Each block connects with others.

- Many small blocks build a large structure.

Angular apps are built by combining many small UI blocks (components).

Not one big HTML file.

Not one big JavaScript file.

- Small blocks.
- Connected together.

## What Problem It Solves in Angular 13

Without components:

1. UI becomes one large HTML file.

2. Logic becomes one large TypeScript file.

3. Hard to reuse parts.

4. Hard to test.

5. Hard to scale.

6. Hard to maintain in teams.

Components solve:

1. UI modularity

2. Reusability

3. Encapsulation (styles & logic isolated)

4. Separation of concerns

5. Easier debugging

6. Scalable architecture

Angular 13 is built entirely on component architecture.

No component → No Angular UI.
